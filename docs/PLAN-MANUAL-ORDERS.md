# План: Создание своих заказов (ручные заказы)

## Цель
Учёт продаж из любых каналов (не только WB/Ozon/Яндекс) — клиент создаёт заказ вручную, указывает товар, количество, стоимость и источник продажи. Заказы отображаются на странице «Заказы на сборке» наравне с маркетплейсными.

---

## Архитектура

### Модель данных

```
Order (расширение)
├── marketplace: MANUAL | WILDBERRIES | OZON | ...
├── salesSource: string?   // "Авито", "Инстаграм" — только для MANUAL
├── externalId: string     // "0001" — ID заказа, вводит клиент
└── ... (остальные поля без изменений)

SalesSource (новая таблица)
├── id: uuid
├── userId: string
├── name: string           // "Авито" (нормализовано: первая буква заглавная)
└── createdAt: datetime
```

**Уникальность:** `[userId, marketplace, externalId]` — для MANUAL + externalId="0001" заказ уникален.

### API

| Метод | Endpoint | Назначение |
|-------|----------|------------|
| POST | `/api/orders` | Создание ручного заказа |
| GET | `/api/products/search?q=` | Поиск товара по id/артикулу/названию (autocomplete) |
| GET | `/api/sales-sources` | Список источников продаж пользователя (autocomplete) |
| POST | `/api/sales-sources` | Сохранение нового источника (upsert по имени) |

### UI

- **Кнопка** «Создать свой заказ» — справа от заголовка «Список заказов на сборке»
- **Модальное окно** с формой:
  - Поле поиска товара (id/арт/название) + выпадающий список совпадений
  - Количество
  - Стоимость
  - Сумма (авто: кол-во × стоимость)
  - Источник продажи (input + autocomplete из сохранённых)
- **Нормализация:** «авито» → «Авито» (первая буква заглавная)
- **Скрыть:** кнопки «Синхронизировать» и «Все заказы»

## Детали реализации

### Нормализация источника продажи
```ts
function normalizeSalesSource(name: string): string {
  const trimmed = name.trim()
  if (!trimmed) return trimmed
  return trimmed.charAt(0).toUpperCase() + trimmed.slice(1).toLowerCase()
}
// "авито" → "Авито", "ИНСТАГРАМ" → "Инстаграм"
```

### Поиск товаров
- Debounce 300ms
- Поиск по: `id` (точное), `article` (ILIKE), `title` (ILIKE)
- Лимит 10 результатов
- Только активные товары (не архив)

### Уникальность externalId
- Для MANUAL: `[userId, MANUAL, externalId]` — уникально
- Клиент вводит "0001" — если уже есть, вернуть ошибку

### Резерв остатка
- При создании ручного заказа: списывать остаток с «Мой склад» (опционально в шаге 4 или позже)

## Масштабируемость

- **Источники продаж** — отдельная таблица, легко добавить аналитику по каналам
- **Ручные заказы** — тот же Order/OrderItem, единая модель для отчётов
- **Поиск товаров** — при росте каталога можно добавить полнотекстовый поиск (pg_trgm, Elasticsearch)

## Риски и митигация

| Риск | Митигация |
|------|------------|
| Дубликат externalId | Уникальный индекс, понятная ошибка |
| Товар не найден | Валидация productId, сообщение «Товар не найден» |
| Пустой источник | Обязательное поле, валидация |
