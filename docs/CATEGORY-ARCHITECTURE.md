# Архитектура поля «Категория» в карточке товара

## Проблема

Ozon требует обязательные поля **Категория** (description_category_id) и **Тип** (type_id). Ozon требует выбор категории **третьего уровня** (например: Хобби и творчество → Материал для рукоделия → Бусина). Захардкоженные 17028922/91565 подходят не для всех товаров и приводят к ошибкам выгрузки.

## Принципы решения

1. **Категории маркетплейс-специфичны** — Ozon и WB имеют разные справочники. Не унифицируем.
2. **Храним ID + путь для отображения** — ozonCategoryId, ozonTypeId (для API), ozonCategoryPath (для UI).
3. **При импорте** — заполняем из данных маркета, если API возвращает категорию.
4. **При создании с нуля** — выбор из дерева Ozon (подгрузка через API).

## Модель данных

| Поле | Тип | Назначение |
|------|-----|------------|
| ozonCategoryId | Int? | description_category_id для Ozon API |
| ozonTypeId | Int? | type_id для Ozon API |
| ozonCategoryPath | String? | Путь для отображения: "Хобби и творчество > Материал для рукоделия > Бусина" |

WB: отдельная структура (subject_id и т.п.) — фаза 2.

## UI

- **Расположение:** после поля «Артикул» в блоке «Основное».
- **Отображение:** путь категории или «Не выбрана» + кнопка «Выбрать категорию».
- **Модальное окно:** дерево категорий Ozon (expand/collapse), поиск. Выбирать можно только узлы с type_id (третий уровень).
- **Валидация:** для выгрузки на Ozon категория обязательна (добавить в validateProductForOzon).

## API

- `GET /api/marketplaces/ozon/categories` — дерево категорий (уже есть).
- При выборе сохраняем ozonCategoryId, ozonTypeId, ozonCategoryPath через PATCH /api/products/:id.

## Импорт

- Ozon v3/product/info/list: при наличии description_category_id и type_id в ответе — сохраняем при создании товара (реализовано).
- WB: при импорте subject_id → wbCategoryId (фаза 2).
