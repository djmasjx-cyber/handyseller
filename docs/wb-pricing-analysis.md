# Анализ ценообразования WB и решение по интеграции

## Текущее состояние в HandySeller

### Что делаем сейчас
- **API:** `discounts-prices-api.wildberries.ru/public/api/v1/prices` (POST)
- **Поля:** `nmId`, `price`, `discount` (всегда 0)
- **Модель:** одна цена `price` в рублях → отправляем как «цена до скидки» = «цена со скидкой»

### Ограничения
1. **Скидка продавца** — не передаём (discount = 0)
2. **Скидка WB Клуба** — не поддерживается
3. **Скидка WB** — задаётся в ЛК WB, через API отдельно
4. **Минимальная цена** — не поддерживается
5. **Участие в акциях** — не поддерживается

## Что видит продавец в ЛК WB (по скриншоту)

| Поле | Описание | API |
|------|----------|-----|
| Цена продавца до скидки | Базовая цена | `price` в v1 |
| Скидка продавца | % скидки | `discount` в v1 |
| Цена со скидкой | Цена после скидки | Рассчитывается WB |

| для участия в акции | Промо-цена | В привязке к акциям |
| Минимальная цена | Минимум для автоакций | Отдельный API |

## WB API: цены и скидки (по документации)

### 1. Prices API v1 (текущий)
- **URL:** `discounts-prices-api.wildberries.ru/public/api/v1/prices`
- **POST:** `[{ nmId, price, discount }]`
- **price** — базовая цена (рубли, целое)
- **discount** — скидка продавца, %
- **Формат:** `price` = цена до скидки; WB сам считает «Цену со скидкой»

### 2. Prices API v2 (расширенный)
- **POST /api/v2/upload/task** — загрузка цен и скидок (асинхронно)
- **POST /api/v2/upload/task/size** — цены по размерам
- **POST /api/v2/upload/task/club-discount** — скидка WB Клуба
- **Ограничения:** 10 запросов / 6 сек, до 1000 товаров за запрос
- **Статус:** `GET /api/v2/history/tasks`, `GET /api/v2/buffer/tasks`

### 3. Получение цен
- **GET /api/v2/list/goods/filter** — товары с ценами
- **GET /api/v2/list/goods/size/nm** — размеры и цены
- **December 2024:** добавлены `clubDiscount`, `clubDiscountedPrice`

### 4. Важные моменты
- **30.01.2025:** base URL перешёл с `suppliers-api` на `content-api` / `discounts-prices-api`
- **Сентябрь 2025:** изменения в рекламном API, не в ценах
- **Rate limit:** 10 запросов / 6 сек на группу методов цен

## Варианты решения

### Вариант A: Расширить текущую модель

**Добавить в каноническую модель:**

```ts
// canonical-product.types.ts
price: number;           // Цена продавца до скидки (рубли)
seller_discount?: number;  // Скидка продавца, % (0–100)
club_discount?: number;    // Скидка WB Клуба, %
minimum_price?: number;    // Минимальная цена для автоакций
```

**Плюсы:**
- Полный контроль над ценами и скидками
- Поддержка всех полей из ЛК WB

**Минусы:**
- Сложная UI: много полей
- Нужно мигрировать на v2 API (асинхронная загрузка)
- Скидка WB и участие в акциях — отдельные API

**Оценка трудозатрат:** 3–5 дней (модель, API v2, UI, тесты)

---

### Вариант B: Упрощённое расширение (только скидка продавца)

**Добавить:**
- `seller_discount` — процент скидки продавца
- `price` — базовая цена
- WB: `price` = цена до скидки, `discount` = скидка

**Плюсы:**
- Минимальные изменения
- Остаётся v1 API

**Минусы:**
- Нет WB Клуба, минимальной цены, акций

**Оценка:** 0.5–1 день

---

### Вариант C: Отказаться от своего ценообразования

**Логика:**
- Не управлять ценами в HandySeller
- Цены задаются только в ЛК WB
- HandySeller — только контент, остатки, заказы

**Плюсы:**
- Нет риска конфликтов с WB
- ЛК WB — источник истины по ценам

**Минусы:**
- Нет единой цены для всех маркетплейсов
- Нет автоматической синхронизации цен

**Оценка:** 0.5 дня (отключить/обернуть вызовы setPrice)

## Рекомендация

1. **Краткосрочно (Вариант B):**  
   Добавить скидку продавца. Это уже поддерживается v1: `discount` передаётся, но сейчас всегда 0. Оценка: ~1 день.

2. **Среднесрочно:**  
   Изучить v2 API и Swagger для полного понимания:
   - `POST /api/v2/upload/task` — схема запроса
   - `POST /api/v2/upload/task/club-discount` — схема
   - Ограничения и статусы задач

3. **Решение по полному ценообразованию:**  
   - Если нужны WB Клуб, минимальная цена и акции — реализовывать Вариант A.  
   - Если достаточно базовой цены и скидки — Вариант B.  
   - Если приоритет — простота и минимум рисков — Вариант C.

## Примечание по формату цены

При импорте используется `price / 100` (копейки → рубли). При установке — `Math.round(price)` (рубли). Стоит сверить с актуальной документацией WB: в разных источниках указывают и рубли, и копейки.
