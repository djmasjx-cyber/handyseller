# Спецификация: Централизованное управление остатками

## 1.1 Назначение

Разработка системы централизованного управления остатками товаров для хендмейтеров, которая автоматически синхронизирует изменения остатков со всеми подключенными маркетплейсами.

## 1.2 Бизнес-цель

Упростить управление запасами для хендмейтеров, исключить ситуацию с продажей отсутствующих товаров и автоматизировать процесс обновления остатков на всех площадках.

---

## 2. Функциональные требования

### 2.1 Управление остатками в приложении

| Требование | Статус |
|------------|--------|
| Хранение единого остатка для каждого товара в центральной БД | ✓ Product.stock |
| Ручное изменение остатков через интерфейс | ✓ replenish |
| Пакетное обновление остатков для нескольких товаров | — |
| История изменений (время, причина, пользователь) | ✓ StockLog |

### 2.2 Синхронизация с маркетплейсами

| Требование | Статус |
|------------|--------|
| Автоматическая синхронизация при изменении остатка | ✓ EventEmitter + StockSyncListener |
| Автоматическая синхронизация при изменении цены | ✓ product.sync.changed + StockSyncListener |
| Синхронизация описания, названия и атрибутов на маркеты | ✓ product.sync.changed при изменении любых полей карточки |
| Двусторонняя синхронизация (учёт продаж) | ✓ Order sync + reserve/release |
| Поддержка нескольких маркетплейсов | ✓ WB, Ozon (частично) |
| Очередь синхронизации | — (в планах) |

### 2.3 Обработка продаж

| Требование | Статус |
|------------|--------|
| Мгновенное уменьшение остатка при продаже | ✓ reserve в OrdersService |
| Распределение остатка между площадками | — (единый остаток) |
| Учёт возвратов и отмен заказов | ✓ release при CANCELLED |
| Частичные отгрузки | — |

### 2.4 Добавление товаров

| Требование | Статус |
|------------|--------|
| Увеличение остатка на всех маркетах при добавлении | — (через Sync) |
| Создание товара с автопубликацией | ✓ uploadProduct (WB) |
| Клонирование товаров | — |

---

## 3. Технические требования

| Требование | Реализация |
|------------|------------|
| Централизованная БД остатков | PostgreSQL + Prisma |
| API для интеграций | NestJS REST API |
| Сервис синхронизации | StockSyncListener (в рамках API) |
| Очередь сообщений | — (EventEmitter, в планах Redis/Bull) |

---

## 4. Алгоритм синхронизации (п.4.1)

```
При изменении остатка или цены в приложении:
1. Обновить данные в БД                  ✓ StockService.change / ProductsService.update
2. Событие product.sync.changed          ✓ EventEmitter
3. StockSyncListener → syncProducts      ✓ Остаток + цена на все маркеты (WB, Ozon, …)
4. Зафиксировать результат               ✓ lastSyncAt, lastError
```
